# Demo

The shortest MemoryOS demo proves persistent cognition.

## Goal

Show that an AI agent can remember, reason, and dream using MemoryOS while durable state connects to 0G through Adapters.

## Flow

1. Create or select an Agent.
2. Store a Memory.
3. Retrieve the Memory.
4. Ask the Agent a question that requires the stored Memory.
5. Run a Dream cycle to consolidate Memory.
6. Install or call an atomic Module when needed.
7. Show the 0G Storage reference for durable Memory.

## What The Demo Must Prove

- MemoryOS is the persistent brain.
- Brain is mandatory.
- LangGraph Runtime is optional.
- Modules are installable capabilities, not agent state.
- 0G is infrastructure, not business logic.
- Brain and Runtime use Adapters for 0G access.

## What The Demo Must Not Become

- A landing page tour
- A blockchain product demo
- An orchestration product demo
- A social product demo
- A token product demo

## Demo Sentence

MemoryOS gives this Agent a brain that survives the session, reasons from stored knowledge, consolidates what it learns, and persists durable state through 0G-backed Adapters.
